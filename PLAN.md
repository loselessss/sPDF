# PDF 편집기 설계 계획서 (v1)

작성일: 2026-07-15 · 위치: `C:\PDF Editor`

**확정된 사항**: UI 한국어 고정 · 본인용(설치본 불필요) · 탐색기 연동은
레지스트리 등록 방식(§8) · OCR은 RapidOCR 기본 + Claude API 옵션(§3.3)

## 1. 목표

가볍고 단순한 데스크톱 PDF 편집기. 핵심 4기능:

1. **페이지 미리보기** — 썸네일 사이드바 + 메인 뷰(줌/스크롤)
2. **OCR** — 스캔본 PDF에 텍스트 레이어를 입혀 "검색/복사 가능한 PDF"로 변환
3. **복사·붙여넣기** — 마우스 드래그로 텍스트 선택 → Ctrl+C
4. **내용 편집(텍스트만)** — 기존 글자를 클릭해서 고치기

## 2. 기술 스택

| 역할 | 선택 | 이유 |
|---|---|---|
| GUI | **PyQt5** | BandWagon과 동일 — 코드 패턴/노하우 재사용 |
| PDF 엔진 | **PyMuPDF (fitz)** | 렌더링·텍스트 추출·편집·저장을 하나로 해결. pip 설치만으로 끝 |
| OCR (기본) | **RapidOCR** (PaddleOCR 모델, ONNX) | pip 설치만으로 끝(별도 설치 프로그램 불필요), 한글 인식 품질이 Tesseract보다 좋음 |
| OCR (옵션1) | **PaddleOCR-VL** (로컬 LLM, 약 1B) | 어려운 페이지용. 오프라인·무료, 단 모델 수 GB + CPU에선 느림 |
| OCR (옵션2) | **Claude API** (vision) | 손글씨 등 최후 수단. API 키 필요, 페이지당 과금 |

의존성: `pip install PyQt5 PyMuPDF rapidocr-onnxruntime` (+ 옵션: `anthropic`). 현재 PC에 모두 미설치 — 첫 구현 때 같이 설치.
만약 Python 3.14에서 onnxruntime 휠이 아직 없으면 Tesseract로 폴백(§6-1의 원안).

## 3. 기능 설계

### 3.1 페이지 미리보기
- 왼쪽: 썸네일 리스트(레이지 렌더링 — 보이는 것만 렌더, 메모리 절약)
- 오른쪽: 메인 페이지 뷰. 줌(Ctrl+휠), 페이지 이동(PgUp/PgDn, 썸네일 클릭)
- 렌더 캐시는 현재 페이지 ± 2페이지만 유지 (대용량 PDF에서 RAM 폭주 방지)

### 3.2 텍스트 선택·복사
- PyMuPDF의 단어 단위 좌표(`page.get_text("words")`)로 드래그 영역과 교차하는 단어를 하이라이트
- Ctrl+C → 클립보드. 스캔본(텍스트 레이어 없음)은 OCR 후에 가능해짐
- "붙여넣기"는 §3.4 편집 모드의 텍스트 입력으로 처리 (PDF에 자유 붙여넣기는 곧 텍스트 삽입)

### 3.3 OCR — 2단계 구조
엔진은 어댑터로 추상화해서 교체/추가 가능하게 설계.

**1단계 — 로컬 (RapidOCR, 기본)**
- 페이지 단위 또는 전체 문서 실행 (백그라운드 스레드, 진행바 표시)
- 단어별 좌표(바운딩 박스)를 얻어 **보이지 않는 텍스트 레이어**로 삽입
  → 원본 외관 그대로, 검색/복사 가능해짐
- 언어: 한국어+영어 동시 인식. 오프라인·무료

**2단계 — 로컬 AI LLM (PaddleOCR-VL, 옵션)**
- 소형(약 1B 파라미터) 문서 특화 vision-LLM. 한국어 포함 109개 언어,
  표/수식/복잡한 레이아웃에 강하고 **좌표도 주므로** 텍스트 레이어 삽입에
  그대로 사용 가능. 오프라인·무료
- 비용: 모델 다운로드 수 GB + 실행 시 RAM 수 GB. GPU 없으면 페이지당
  수십 초 수준으로 느림 → RapidOCR 대체가 아니라 "어려운 페이지용 옵션"
- 설정에서 활성화 시에만 모델 다운로드 (미설치면 메뉴 비활성)

**3단계 — 클라우드 AI (Claude API, 옵션)**
- 손글씨 등 로컬 AI로도 안 되는 최후 수단. vision 모델은 좌표를 정밀하게
  못 주므로 로컬 OCR 좌표에 AI가 읽은 텍스트를 입혀 교정하거나,
  결과를 사이드 패널에 띄워 복사용으로 제공
- 설정에 API 키 입력란. 키 없으면 메뉴 비활성
- 인터넷 필요, 페이지당 소액 과금 — 실행 전 페이지 수 확인 대화상자

### 3.4 텍스트 편집 — 방식과 한계 (중요)
PDF는 워드가 아니라 "글자를 좌표에 찍어놓은" 포맷이라 진짜 인라인 편집은 없음. 현실적인 방식:

- 편집 모드에서 텍스트 블록 클릭 → 해당 줄의 글자를 지우고(redaction) 같은 자리에 새 텍스트를 다시 씀
- **폰트**: 원본 폰트 재사용이 안 되는 경우가 많아 대체 폰트(맑은 고딕 등)로 다시 씀 → 모양이 미묘하게 달라질 수 있음. 크기·색은 원본에서 읽어와 맞춤
- **리플로우 없음**: 글자가 길어져도 다음 줄로 안 밀림. 해당 줄 안에서만 교체 (넘치면 자간/크기 자동 축소 또는 경고)

**스캔본 편집** (지원함):
- OCR을 먼저 실행해 단어 좌표를 확보한 뒤, 편집할 단어 클릭 →
  ① 그 영역을 **주변에서 샘플링한 배경색으로 칠해** 원본 글자를 지우고
  ② 같은 자리에 새 텍스트를 실제 텍스트 객체로 그림(검색도 됨)
- 배경이 균일한 문서는 티가 거의 안 나고, 노이즈 많은 스캔은 지운 자리가
  약간 표시날 수 있음 (필요시 cv2 inpaint로 개선 여지)
- OCR 없이도 임의 위치에 "텍스트 박스"(배경 사각형 + 새 글자)를 얹는
  자유 편집 모드 제공

### 3.5 추가 제안 (요청하신 "또 필요한 게 있을까")
반쯤 공짜로 얻을 수 있어서 넣기를 권장:
- **페이지 조작**: 회전 / 삭제 / 순서 변경(썸네일 드래그) / 다른 PDF 병합 / 추출(분할)
- **텍스트 검색** (Ctrl+F) — 복사 기능과 같은 텍스트 레이어를 쓰므로 거의 공짜
- **다른 이름으로 저장** + 저장 전 원본 자동 백업
- **최근 파일 메뉴**, 드래그&드롭으로 열기

보류 권장 (필요해지면 그때):
- 주석(하이라이트/메모), 폼 채우기, 전자서명, 이미지 삽입, 암호 설정

## 4. 아키텍처

BandWagon과 같은 "메인윈도우 + 믹스인 + Qt 비의존 코어" 구조:

```
pdfeditor/
  app.py        # MainWindow — 믹스인 조립
  viewer.py     # ViewerMixin — 썸네일/메인뷰/줌/페이지 이동
  textsel.py    # TextSelectMixin — 선택/복사/검색
  editing.py    # EditMixin — 텍스트 편집 + undo/redo
  ocr.py        # OCR 워커(QThread) + 엔진 어댑터 (Qt 시그널만 의존)
  core.py       # PyMuPDF 래핑 — 열기/저장/렌더/텍스트추출 (Qt 비의존)
  widgets.py    # PageView, ThumbList
  i18n.py       # ko/en (BandWagon 방식 그대로)
run.py / run.pyw
```

- **undo/redo**: BandWagon처럼 "원본 + 연산 리스트 재생" 방식. 편집 연산(텍스트 교체, 페이지 삭제 등)을 기록하고 되돌리기 시 원본에서 재적용
- **저장**: 항상 새 파일로 쓰고 성공 시 교체 (원본 파손 방지). incremental save는 쓰지 않음

## 6. 결정 사항 (확정)

1. **OCR 엔진**: RapidOCR 기본 + Claude API 옵션 (§3.3).
   당초 후보였던 Tesseract는 별도 설치 프로그램이 필요하고 한글 품질도
   RapidOCR보다 떨어져 제외. 단, Python 3.14용 onnxruntime 휠이 없으면
   Tesseract로 폴백.
2. **UI 언어**: 한국어 고정 (i18n 모듈 생략 — 문자열 직접 기입)
3. **배포**: 본인용. exe 빌드 없이 `run.pyw` + 탐색기 연동(§8)
4. **추가 기능**: §3.5 권장 목록 포함 (페이지 조작·검색·백업·최근 파일)

## 7. 리스크

- **텍스트 편집 품질**: 폰트 대체로 인한 외관 변화가 이 도구의 가장 큰 한계. "완벽 재현"이 아니라 "내용 수정"이 목표임을 전제
- **암호화 PDF**: 열 때 비밀번호 입력만 지원, 편집 제한 플래그는 존중
- **초대형 PDF**(수백 페이지 스캔본): 레이지 렌더링 + 캐시 제한으로 대응 (§3.1)

## 8. 탐색기 연동 (exe 없이)

PDF를 더블클릭하거나 "연결 프로그램"으로 열려면 실행 파일을 등록해야
하지만, **exe를 만들 필요는 없다** — 현재 사용자(HKCU) 레지스트리에
`pythonw.exe "C:\PDF Editor\run.pyw" "%1"` 형태의 연결 항목을 등록하면
탐색기 우클릭 → 연결 프로그램 목록에 "PDF Editor"가 나타난다.

- 프로젝트에 `register_filetype.py`(등록/해제 스크립트) 포함 — 한 번 실행
- 기본 PDF 뷰어를 뺏지 않도록 **"연결 프로그램" 후보로만 등록**이 기본,
  기본 프로그램 지정은 사용자가 Windows 설정에서 직접 선택
- `run.pyw`는 이미 인자로 파일 경로를 받는 구조(§5 v0.1)라 추가 작업 없음
